I could say I am also pro "reading tests, and less reading code". What is good code anyway?

- **Test driven**, this doesn't mean you write the tests and then write the function. You can write the function first and then write the tests. But do write the tests for everything possible case.
- **Declarative**, where applicable.
- **Self explanatory**, names of functions and variables should be self explanatory. This is basically the hardest part of software engineering when you reach the seniority. Now made much easier thanks to LLM advancements AI.
- **Focus before features**, or simply put YAGNI
